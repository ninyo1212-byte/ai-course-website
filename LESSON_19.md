## מושגים מרכזיים

- כלי אוטומציה (Automation Platform): מערכת שמאפשרת לחבר בין אפליקציות ושירותים ולבנות תהליכים אוטומטיים.
- Make: כלי אוטומציה ויזואלי לבניית תרחישים שמחברים בין אפליקציות, נתונים ופעולות.
- n8n: כלי אוטומציה ויזואלי וגמיש לבניית Workflows, עם אפשרות לעבודה בענן או בהתקנה עצמית, לפי הצורך.
- תרחיש (Scenario): השם המקובל ב-Make לאוטומציה שמורכבת מטריגר ומודולים.
- זרימת עבודה (Workflow): השם המקובל ב-n8n לאוטומציה שמורכבת מטריגר וצמתים.
- מודול (Module): רכיב בתוך Make שמבצע פעולה אחת, למשל קבלת טופס, שמירת שורה או שליחת בקשה.
- צומת (Node): רכיב בתוך n8n שמבצע פעולה אחת או מפעיל חלק בזרימה.
- מחבר (Connector / Integration): חיבור מוכן לאפליקציה או שירות, למשל Google Sheets, Gmail, Airtable או OpenAI.

## הסבר מרכזי

בשיעורים 16-18 למדנו לתכנן אוטומציה: מה מתחיל אותה, מה היא עושה, איפה AI משתלב ואיפה אדם בודק.

עכשיו אנחנו מתחילים להכיר כלים שיכולים לבנות אוטומציות כאלה בפועל.

Make ו-n8n הם שני כלים נפוצים לבניית אוטומציות. שניהם מאפשרים לחבר בין מערכות בלי להתחיל מקוד מלא. אבל הם לא אותו דבר, ולא בוחרים ביניהם רק לפי מה שפופולרי.

השאלה המקצועית היא:

"איזה כלי מתאים לתהליך, לרמת הידע, ללקוח ולתחזוקה?"

### מה Make עושה

Make מאפשר לבנות תרחישים ויזואליים. תרחיש נקרא Scenario.

בתרחיש יש בדרך כלל:

1. טריגר שמתחיל את התהליך.
2. מודולים שמבצעים פעולות.
3. חיבורים בין מודולים.
4. תנאים, סינונים או מסלולים לפי הצורך.

לדוגמה:

- טופס חדש התקבל.
- Make שומר את הפרטים בגיליון.
- Make שולח את הטקסט לכלי AI.
- Make שומר את הסיכום.
- Make שולח התראה לבעל העסק.

Make מתאים במיוחד כאשר רוצים לבנות אוטומציה ויזואלית במהירות, לעבוד עם הרבה אפליקציות מוכנות, ולהתחיל בלי הרבה ידע טכני.

### מה n8n עושה

n8n מאפשר לבנות זרימות עבודה שנקראות Workflows.

ב-Workflow יש:

1. טריגר או צומת התחלה.
2. Nodes שמבצעים פעולות.
3. חיבורים בין הצמתים.
4. תנאים, פיצולים, טיפול בשגיאות ועבודה עם נתונים.

לדוגמה:

- Webhook מקבל פנייה חדשה.
- Node שומר את הנתונים.
- Node שולח בקשה ל-AI.
- Node בודק אם חסרים פרטים.
- Node מעביר את הפנייה למסלול המתאים.

n8n מתאים במיוחד כאשר רוצים יותר שליטה וגמישות, כאשר יש צורך בעבודה טכנית יותר, או כאשר חשוב להבין ולעצב את זרימת הנתונים בצורה מפורטת.

### Scenario מול Workflow

המונחים שונים, אבל הרעיון דומה.

| מושג | Make | n8n |
|---|---|---|
| שם האוטומציה | Scenario | Workflow |
| רכיב פעולה | Module | Node |
| תחילת תהליך | Trigger Module | Trigger Node |
| פעולה רגילה | Module | Action Node |
| פיצול לפי תנאי | Filter / Router | IF / Switch / Branch |

חשוב לא להתבלבל מהמילים. בשני הכלים אנחנו עדיין חושבים באותה צורה:

טריגר -> פעולה -> תנאי -> פעולה -> בקרה.

### מתי לבחור Make

Make יכול להתאים כאשר:

- רוצים להתחיל מהר.
- הלקוח צריך פתרון ויזואלי שקל להסביר.
- רוב החיבורים הם לאפליקציות מוכרות.
- התהליך לא דורש לוגיקה מורכבת מדי.
- חשוב לבנות אבטיפוס מהר.
- מי שמתחזק את התהליך לא טכני מאוד.

לדוגמה:

פנייה מטופס -> שמירה בגיליון -> סיכום AI -> התראה במייל.

זה תרחיש ש-Make יכול להתאים לו מאוד.

### מתי לבחור n8n

n8n יכול להתאים כאשר:

- רוצים יותר שליטה בזרימת הנתונים.
- יש צורך ב-Webhook, HTTP Request או עיבוד נתונים גמיש.
- יש צוות טכני יותר.
- רוצים אפשרות להתקנה עצמית או שליטה בסביבה.
- התהליך כולל תנאים, פיצולים או טיפול בשגיאות בצורה משמעותית.
- יש צורך לחבר מערכות פחות סטנדרטיות.

לדוגמה:

Webhook מקבל פנייה -> בדיקת שדות -> קריאה ל-AI -> פיצול לפי סטטוס -> שמירה -> התראה -> טיפול בשגיאות.

### ההבדל העקרוני

Make נוטה להיות נוח מאוד להתחלה ויזואלית מהירה.

n8n נוטה לתת יותר תחושת "זרימת עבודה טכנית" עם שליטה גבוהה יותר בפרטים.

זו לא תחרות של "מי טוב יותר". אלה כלים שונים שמתאימים למצבים שונים.

### איך לבחור בלי להסתבך

אפשר להשתמש בשאלות הבאות:

1. האם האוטומציה פשוטה יחסית או מורכבת?
2. האם הלקוח או הצוות צריכים להבין ולתחזק אותה בעצמם?
3. האם רוב החיבורים קיימים ככלים מוכנים?
4. האם נדרש Webhook או API מותאם?
5. האם יש דרישות פרטיות, אחסון או שליטה בסביבה?
6. האם חשוב לבנות מהר או לשלוט לעומק?

בשלב הלימוד, חשוב יותר להבין את העקרונות מאשר "להינעל" על כלי אחד.

## דוגמה מעשית

נמשיך עם סטודיו לעיצוב פנים.

האוטומציה:

פנייה חדשה מטופס נכנסת, נשמרת, נשלחת לסיכום עם AI, והתוצאה עוברת לבעלת העסק לבדיקה.

איך זה נראה ברמת עיקרון:

| שלב | Make | n8n |
|---|---|---|
| קבלת פנייה | Module של טופס או Webhook | Trigger Node או Webhook Node |
| שמירת פרטים | Module של Google Sheets / Airtable | Node של Google Sheets / Airtable |
| סיכום עם AI | Module של OpenAI / כלי AI | Node של OpenAI / HTTP Request |
| בדיקת חסרים | Filter / Router | IF / Switch |
| התראה לבעלת העסק | Email / Gmail / Slack Module | Email / Gmail / Slack Node |

אם רוצים להתחיל מהר עם טופס וגיליון, Make יכול להיות בחירה נוחה.

אם רוצים לבנות זרימה גמישה יותר, לקבל Webhook מותאם, לבדוק תנאים מורכבים ולשלוט טוב יותר בנתונים, n8n יכול להיות מתאים יותר.

בשלב הראשון של הקורס, הבחירה לא חייבת להיות סופית. אפשר לתכנן את האוטומציה כך שתהיה ברורה בכל כלי.
