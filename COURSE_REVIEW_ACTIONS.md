# מפת פעולות בעקבות ביקורת הקורס

## מטרת המסמך

מסמך זה מסדר את הביקורת החיצונית לרשימת החלטות זהירה עבור המשך פיתוח הקורס.

המטרה היא לחזק את הרמה המעשית והמקצועית של הקורס בלי לשכתב את 36 השיעורים, בלי לשנות את סדר הלמידה ובלי להפוך את הקורס למערכת אינסופית של כלים ונושאים.

המסמך מבוסס בשלב זה על נושאי הביקורת שסופקו: תרגול מעשי, הערכה ומדדי הצלחה, אבטחה וסיכונים, רגולציה ואתיקה, תוצרי לקוח, עבודה מתקדמת עם Codex / Claude Code / GitHub ועדכון טכנולוגי מתמשך. לא בוצע במסגרת מסמך זה מחקר מקורות רחב.

---

## עקרונות קבלת החלטות

- שומרים על 7 החלקים, 10 הנושאים ו־36 השיעורים הקיימים.
- מעדיפים מסמכים נלווים, תבניות ותרגולים על פני שינוי שיעורים.
- כל תוספת צריכה לחזק תוצר למידה קיים ולא לפתוח מסלול חדש.
- עקרונות קבועים קודמים לכלים משתנים.
- מידע בטיחותי, רגולטורי או תלוי־גרסה ייכנס רק לאחר אימות מול מקורות רשמיים.
- כל שינוי תוכן, מבנה או טכנולוגיה דורש משימה ואישור נפרדים.

## סדר עדיפויות מומלץ

1. להכין חבילת תרגולים קצרה לפי 7 החלקים.
2. להכין Rubric אחד לפרויקט המרכזי ו־Checklist מוכנות לכל חלק.
3. להכין חבילת תוצרי לקוח בסיסית.
4. לתכנן מעבדה נלווית אחת ל־Codex / Claude Code / GitHub, בכפוף לבדיקה רשמית עדכנית.
5. לבצע מחקר נפרד ומאושר לפני פרסום חומר על אבטחה, רגולציה, פרטיות או Evaluation טכני.
6. לבחון את התוספות עם שימוש אמיתי לפני החלטה על גרסה 2.

## המלצה מרכזית

גרסה 1.1 צריכה לחזק **ביצוע, הערכה ותוצר מקצועי** באמצעות חומרים נלווים קטנים: 7 תרגולים, Checklist לכל חלק, Rubric אחד וחבילת תוצרי לקוח.

אין צורך לפתוח מחדש את 36 השיעורים. אבטחה בסיסית וסביבת עבודה מתקדמת עם כלי פיתוח יכולות להיכלל רק כתוספות ממוקדות ולאחר בדיקה מתאימה. רגולציה, מסלולים מתקדמים ותלות ב־Frameworks יישארו לגרסה 2 או מחוץ להיקף עד לקבלת החלטה נפרדת.
