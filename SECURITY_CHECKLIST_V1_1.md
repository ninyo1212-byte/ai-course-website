# Checklist אבטחה בסיסי — V1.1

## מטרה ומועד שימוש

- [ ] עוברים על ה־Checklist לפני דמו, מסירה או עבודה עם מידע וגישה של לקוח.

## 1. מידע רגיש של לקוח

- [ ] נאסף רק המידע הדרוש לפרויקט.
- [ ] מידע רגיש אינו מועתק לכלי, קובץ או אדם שאינם נדרשים לעבודה.
- [ ] ברור היכן המידע נשמר ומי יכול לגשת אליו.

## 2. API Keys ו־Secrets

- [ ] API Keys, סיסמאות ו־Secrets אינם מופיעים בקוד, במסמכים, בצילומי מסך או במאגר Git.
- [ ] ה־Secrets נשמרים במקום ייעודי, כגון Environment Variables או מנהל סודות.
- [ ] Secret שנחשף מבוטל ומוחלף.

## 3. הרשאות מינימליות

- [ ] לכל משתמש, כלי וחיבור ניתנה רק ההרשאה הנדרשת למשימה.
- [ ] לא נעשה שימוש בחשבון מנהל כאשר הרשאה מצומצמת מספיקה.
- [ ] גישות זמניות מוגבלות בזמן או מוסרות כשאינן נחוצות.

## 4. אישור אדם לפני פעולה רגישה

- [ ] שליחה, מחיקה, תשלום, פרסום או שינוי בלתי הפיך דורשים אישור אדם.
- [ ] נקודת האישור ברורה ואינה ניתנת לדילוג בטעות.
- [ ] האדם המאשר רואה מה עומד להתבצע לפני האישור.

## 5. בדיקת לוגים ומניעת חשיפת מידע

- [ ] הלוגים נבדקו ואינם חושפים Secrets, סיסמאות או מידע רגיש מלא.
- [ ] הודעות שגיאה אינן מציגות מידע פנימי שאינו נחוץ למשתמש.
- [ ] נשמר רק המידע הנחוץ לבדיקה ולטיפול בתקלה.

## 6. נתוני דמה בדמו

- [ ] הדמו משתמש בנתוני דמה ולא במידע אמיתי של לקוח.
- [ ] שמות, כתובות, טלפונים ומזהים בדמו מסומנים כדוגמה.
- [ ] צילומי מסך והקלטות נבדקו לפני שיתוף.

## 7. מסירת גישות וסגירת הרשאות

- [ ] הגישות נמסרות ללקוח בערוץ מתאים ולא בתוך מסמך פתוח או הודעה ציבורית.
- [ ] בעלות על חשבונות, אוטומציות וקבצים מועברת בצורה ברורה.
- [ ] בסיום העבודה מוסרות גישות זמניות ומבוטלים Secrets שאינם נדרשים עוד.

## 8. פעולות בסיסיות במקרה תקלה

- [ ] עוצרים את האוטומציה או הפעולה הבעייתית כדי לצמצם נזק נוסף.
- [ ] מתעדים בקצרה מה קרה, מתי ואיזה מידע או שירות הושפעו.
- [ ] מודיעים לאדם האחראי, מבטלים גישה או Secret שנחשפו, ובודקים לפני הפעלה מחדש.

## גבולות

- [ ] לא רגולציה עמוקה.
- [ ] לא ייעוץ משפטי.
- [ ] לא בדיקות חדירה.
- [ ] לא ארכיטקטורת אבטחה מתקדמת.
- [ ] לא תוכנית תגובה מלאה לאירועים.
