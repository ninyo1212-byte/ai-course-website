# DEPLOYMENT_NOTES.md

# הערות הכנה להעלאת האתר

## 1. מצב הפרויקט לפני העלאה

הפרויקט הוא אתר Next.js מקומי לקורס AI בעברית. האתר כולל מסך פתיחה, מפת קורס, 36 שיעורים מתוך קובצי Markdown, פרויקט מרכזי והתקדמות אישית.

האתר פועל בגישת Desktop-first Responsive. אין בגרסה הראשונה התחברות, משתמשים מרובים, מסד נתונים, שמירה בענן, תשלומים או סוכן AI.

---

## 2. בדיקות חובה לפני העלאה

לפני העלאה יש להריץ מתיקיית הפרויקט:

```bash
npm run lint
npm run build
```

שתי הפקודות צריכות להסתיים בהצלחה לפני שממשיכים לפריסה.

---

## 3. קבצים חשובים שחייבים להיות במאגר

יש לוודא שהמאגר כולל לפחות:

- `package.json`
- `package-lock.json`
- `next.config.ts`
- `tsconfig.json`
- `app/`
- `components/`
- `lib/`
- `START_HERE.md`
- `COURSE_INDEX.md`
- `PROJECT_TRACKER.md`
- `LESSON_01.md` עד `LESSON_36.md`

קובצי ה־Markdown הם מקור האמת לתוכן, ולכן חייבים להיות זמינים גם בזמן הבנייה בפריסה.

---

## 4. משתני סביבה

בגרסה הנוכחית אין צורך במשתני סביבה.

אין להוסיף קובצי סודות, מפתחות API או הגדרות `.env` ללא צורך מפורש ומאושר.

---

## 5. שתי אפשרויות העלאה

### GitHub + Vercel

מעלים את הפרויקט למאגר GitHub ומחברים אותו לפרויקט ב־Vercel. לאחר החיבור, עדכונים לענף הראשי יכולים ליצור פריסת Production, וענפים אחרים יכולים לשמש לפריסות Preview.

### Vercel CLI

אפשר לבצע פריסה ישירה מתיקיית הפרויקט באמצעות כלי הפקודה של Vercel. אפשרות זו מתאימה לפריסה מקומית ישירה, אך אינה מחליפה ניהול מסודר של קוד וגרסאות במאגר.

---

## 6. המלצה

מומלץ להשתמש ב־GitHub + Vercel אם רוצים ניהול מסודר של הקוד, היסטוריית שינויים, פריסות Preview ופריסת Production ברורה.

---

## 7. סיכון חשוב: localStorage

ההתקדמות האישית, סימוני רשימות הבדיקה והטקסט בפרויקט נשמרים ב־`localStorage` של הדפדפן.

מידע זה אינו מסתנכרן בין מכשירים, בין דפדפנים או בין דומיינים שונים. מעבר לכתובת Vercel חדשה עשוי להתחיל מצב התקדמות מקומי נפרד בדפדפן.

זו ההתנהגות המאושרת לגרסה הראשונה, ללא התחברות וללא שמירה בענן.

---

## 8. בדיקות אחרי העלאה

- לפתוח את מסך הפתיחה.
- לפתוח את מפת הקורס.
- לבדוק שיעור 01 ושיעור 36.
- לבדוק מעבר לשיעור הקודם והבא.
- לבדוק טעינה תקינה של תוכן Markdown בפריסה.
- לסמן התקדמות ולרענן את הדפדפן.
- לכתוב טקסט קצר בפרויקט המרכזי ולרענן.
- לפתוח את עמוד ההתקדמות.
- לבדוק תצוגה במחשב ובמסך צר.

לא להוסיף בשלב זה:

- התחברות או ניהול משתמשים.
- מסד נתונים.
- סוכן AI.
- תשלומים או מנויים.

כל אחד מהנושאים האלה ייבחן רק כשלב נפרד, לאחר הגדרה ואישור מפורשים.
