# Checkpoint 04 — סגירת גרסה 1.1

## 1. מטרת גרסה 1.1

גרסה 1.1 נועדה לחזק ביצוע מעשי, בדיקה עצמית ותוצר מקצועי, בלי לשכתב את 36 השיעורים ובלי להרחיב את הקורס למסלול נוסף.

---

## 2. מה נוסף מבחינת תוכן

- `COURSE_REVIEW_ACTIONS.md` מתעד את מיון הביקורת והפעולות האפשריות להמשך.
- שכבת תרגול מתקדמת עם עבודה מבוקרת באמצעות Codex, Claude Code ו־GitHub, תוצרי לקוח ובדיקות איכות.
- תרגיל אופציונלי קצר לכל אחד משבעת חלקי הקורס.
- Checklist אופציונלי למוכנות לפני מעבר מכל חלק לחלק הבא.

---

## 3. איך זה שולב באתר

חומרי גרסה 1.1 שולבו בתוך מפת הקורס בנתיב `/course`, לפי שבעת חלקי הקורס.

בכל חלק מופיע אזור סגור כברירת מחדל בשם „לאחר סיום חלק זה”. האזור מבהיר מתי לבצע אותו, מציג תרגיל מסכם, Checklist, תוצר שצריך להישאר בפרויקט המרכזי וקישור לעמוד הפרויקט.

- חלק 6 כולל תרגול מתקדם עם Codex, Claude Code ו־GitHub.
- חלק 7 כולל תוצרי לקוח: מסמך אפיון, דמו או הסבר פתרון, הצעת מחיר בסיסית, תוכנית מסירה ותחזוקה ותיעוד בדיקות.

קובצי ה־Markdown נשארו מקור האמת לתוכן.

---

## 4. למה `/practice` הוסר

העמוד `/practice` הציג את חומרי גרסה 1.1 כעמוד מסמכים מרוכז. תצוגה זו לא הבהירה ללומד מתי לבצע כל תרגיל, מתי להשתמש ב־Checklist ואיך כל תוצר מתקדם עם הפרויקט המרכזי.

החומרים נשמרו, אך הועברו להקשר הטבעי שלהם בתוך כל חלק במפת הקורס. הנתיב והקישור „תרגול” הוסרו.

---

## 5. מה נבדק ועובד

- `npm run lint` עבר בהצלחה.
- `npm run build` עבר בהצלחה.
- מפת הקורס מציגה שבעה חלקים ו־36 שיעורים, כל שיעור פעם אחת.
- בכל חלק קיים אזור „לאחר סיום חלק זה” עם תרגיל, Checklist ותוצר לפרויקט.
- חלקים 1, 6 ו־7 נבדקו, כולל התרגול המתקדם ותוצרי הלקוח.
- שיעורים 01 ו־36 נפתחים, והניווט קודם/הבא עובד.
- עמודי הפרויקט וההתקדמות נפתחים.
- לא נמצאו אזהרות React או duplicate keys בבדיקה המקומית.

---

## 6. מה לא שונה

- 36 השיעורים וקובצי ה־Markdown שלהם.
- תוכן הקורס, סדר השיעורים ומבנה שבעת החלקים.
- לוגיקת `localStorage` של התקדמות אישית והטקסט בפרויקט.
- עמוד הפרויקט המרכזי ועמוד ההתקדמות.
- אין התחברות, מסד נתונים, שמירה בענן, ציונים, תעודות, תשלומים או סוכן AI באתר.
